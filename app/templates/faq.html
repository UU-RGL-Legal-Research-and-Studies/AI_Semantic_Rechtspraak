Frequently Asked Questions

What is the purpose of JurisAnalyser?

JurisAnalyser is designed to systematically analyze and search case law (ECLIs) from rechtspraak.nl. JurisAnalyser uses both traditional (keywords) and semantic (AI) queries. It helps users efficiently find relevant case law information.

How does JurisAnalyser work?

JurisAnalyser works by uploading legal documents from rechtspraak.nl based on ECLI-numbers. Queries can be performed wit keywords and so-called natural language processing (NLP) and machine learning (AI). As such, the tool can perform both keyword-based searches and semantic searches. Additional benefits are:

  • Users do not have to open each case law document; the relevant paragraphs are immediately visible.
  • The relevant paragraphs can be stored in an Excel file for systematic analysis.
  • Traditional and semantic search results can be combined to verify and cross-check the findings.
  • A link to the original case law is provided if further validation or context is necessary.

How is the data collected?

The tool uses the API (Application Programming Interface) from rechtspraak.nl to fetch data for each ECLI listed in the uploaded Excel file. This data is then stored in the database for analysis. The API request retrieves XML data, which is parsed and is processed for further use.

What is the difference between traditional and semantic queries?

Traditional query:

Traditional queries look for exact matches of the input search terms within the text. Each paragraph of the legal document is scanned for the presence of the search terms. If the search term is found within the paragraph, that paragraph is highlighted and included in the results.

Semantic query:

Semantic queries use so-called cosine similarity to find the most relevant paragraphs that match the semantic meaning of the query. Results are ranked based on similarity scores, and only those above a certain threshold are included in the results. Users can adjust the similarity metrics for the semantic query. Experimenting with the metric is advised to obtain the best results. A 0.5 metric is a good starting point. Semantic queries 'understand' the meaning of the search terms and find relevant results based on context and synonyms. The entire text is encoded using a pre-trained model, and the encoded query is compared against these encodings. Semantic similarity does not generate new content but rather identifies the similarity between the user's query and the existing case law.

How can I view the results?

After the documents are uploaded and a query has been asked, you can view the results on the 'Analysis Results' page. Here, you can browse through the found results and view details. The analysis results can be stored in an Excel file. The entire dataset can be stored in a JSON file for more efficient reusage (i.e. using the dataset again without an API-request to rechtspraak.nl).

How does JurisAnalyser ensure valid results?

JurisAnalyser uses both (a) traditional search methods (keywords) and (b) advanced natural language processing (NLP) and machine learning algorithms to ensure high accuracy. Additionally, the tool:

  • Fetches data directly from rechtspraak.nl, ensuring the information is up-to-date and reliable.
  • Enables users to cross-check traditional and semantic queries, as well as view the original source.
  • All results are distilled directly from paragraphs of the extracted cases. 'Hallucinated text' is therefore not possible.
  • JurisAnalyser is completely open source. All code upon which JurisAnalyser is built, including the search methods, are completely transparent.

What makes JurisAnalyser different from other legal search engines?

JurisAnalyser is different as it:

  • Combines both traditional (keywords) and semantic (AI) queries for comprehensive search results.
  • Creates a faster workflow by making relevant data immediately visible on screen and in Excel.
  • Is completely open source. The query methodologies are transparent which increases the validity of the results.
  • It uses a specific type of AI (semantic similarity) which is reproducable. A similar query in the semantic search and similarity score will produce the same result.
  • Is low-power consuming. Due to efficient code and efficient GPU-usage, JurisAnalyser uses approximately 35-70W/hour. By comparison, when being used, a laptop uses between 50-100W/hour.
